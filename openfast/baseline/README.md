# OpenFAST 5 MW Baseline Files

This repository deliberately does not redistribute the NREL 5 MW baseline
OpenFAST input files. Those files are maintained by the National
Renewable Energy Laboratory and are freely available from the OpenFAST
project's reference test repository on GitHub. Bundling them here would
duplicate the upstream distribution, risk drifting out of sync with NREL's
authoritative versions, and inflate this repository with files that are
not part of the contribution of the accompanying paper.

## Where to obtain the baseline files

The NREL 5 MW reference turbine baseline files are distributed as part of
OpenFAST's regression-test package, available at:

```
https://github.com/OpenFAST/r-test
```

The relevant subdirectory inside that repository is
`glue-codes/openfast/5MW_OC3Spar`, which contains the OC3 Hywind spar
configuration that the paper uses. From the same repository you also need
the shared 5 MW baseline data under `glue-codes/openfast/5MW_Baseline`,
which contains the aerodynamic, hydrodynamic, blade, and tower property
files referenced by the OC3 Hywind input deck.

## Where to place the files locally

Once obtained, copy the entire `5MW_Baseline` directory into this
location inside your local clone of this repository:

```
openfast/baseline/5MW_Baseline/
├── AeroData/
├── Airfoils/
├── HydroData/
├── ServoData/
├── Wind/
├── NRELOffshrBsline5MW_AeroDyn_blade.dat
├── NRELOffshrBsline5MW_BeamDyn.dat
├── NRELOffshrBsline5MW_BeamDyn_Blade.dat
├── NRELOffshrBsline5MW_Blade.dat
├── NRELOffshrBsline5MW_InflowWind_Steady8mps.dat
├── NRELOffshrBsline5MW_InflowWind_Steady12mps.dat
├── NRELOffshrBsline5MW_InflowWind_12mps_old.dat
├── NRELOffshrBsline5MW_Monopile_IEC_Crushing.inp
├── NRELOffshrBsline5MW_ServoDyn_StC.dat
└── ... (other files as distributed by NREL)
```

The `.gitignore` for this repository excludes the entire
`openfast/baseline/5MW_Baseline/` subtree from version control, so the
files you add locally will not be committed accidentally. The customised
case-generation templates that this repository does provide live in
`openfast/templates/nonlinear/` and `openfast/templates/linear/` and
reference the baseline files via paths relative to the OpenFAST run
directory at execution time.

## Wind input regeneration

The original twelve-case workflow used a turbulent wind binary
(`short_turb12mps.bts`) generated by TurbSim, the IEC-Kaimal turbulence
generator distributed alongside OpenFAST. This file is approximately
70 MB and is therefore excluded from version control. Users who wish to
regenerate it should run TurbSim against the input deck
`90m_12mps_twr.inp` from NREL's `5MW_Baseline/Wind/` directory:

```
turbsim 90m_12mps_twr.inp
```

The resulting `90m_12mps_twr.bts` should be renamed to
`short_turb12mps.bts` (or the wind-file references inside the templates
should be updated to point at whichever filename you produce).
Alternatively, the steady-wind input files
`NRELOffshrBsline5MW_InflowWind_Steady8mps.dat` and
`NRELOffshrBsline5MW_InflowWind_Steady12mps.dat` from the NREL baseline
can be used for runs that do not require a turbulent inflow, including
the linearization workflow.

## OpenFAST executable

This repository does not bundle the OpenFAST executable. Users must
install OpenFAST separately and provide the path to their executable in
`configs/openfast_cases.yaml` under the `openfast_exe` field. OpenFAST
binaries for Windows, Linux, and macOS are available from the OpenFAST
GitHub releases page at:

```
https://github.com/OpenFAST/openfast/releases
```

The version used in the paper's runs is OpenFAST 3.x; later versions are
likely to be compatible but have not been independently tested for this
codebase.
